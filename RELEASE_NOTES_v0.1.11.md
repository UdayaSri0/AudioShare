# SynchroSonic v0.1.11

SynchroSonic v0.1.11 is a release-alignment and packaging verification update focused on preparing a clean tagged release after the v0.1.10 release line.

## Highlights

- aligned the project on version `0.1.11` across Cargo workspace metadata, packaging metadata, changelog entries, release notes, and public documentation
- refreshed release-facing documentation so the current release line points to `v0.1.11`
- rechecked the tagged release workflow so GitHub Actions validates the tag against the root workspace version
- preserved the Linux installer artifact path for AppImage, Debian `.deb`, Flatpak bundle, portable tarball, and checksum publication
- kept canonical repository, issue, and release links pointed to `https://github.com/UdayaSri0/AudioShare`

## Fixed

- fixed stale release references from earlier `0.1.9` and `0.1.10` preparation passes
- fixed documentation wording that could make the current release line look older than the latest GitHub release
- confirmed that release artifact verification still requires real installable outputs before publishing
- confirmed that generated release assets should not be replaced by source archives only

## Packaging Scope

The tagged release workflow should publish these assets when the release succeeds:

- `synchrosonic-0.1.11-x86_64.AppImage`
- `synchrosonic_0.1.11_amd64.deb`
- `synchrosonic-0.1.11.flatpak`
- `synchrosonic-0.1.11-linux-x86_64.tar.gz`
- `SHA256SUMS.txt`

If `RELEASE_MANIFEST.json` is generated by the current scripts, include it in the release assets as well.

## Notes

- Flatpak remains a preview/runtime-validation path where host PipeWire CLI access may still affect runtime behaviour.
- AppImage and Debian package signing remain future work unless signing has already been implemented in the repository.
- Signed APT repository publication should remain a separate workflow unless this repository already contains a validated signed publication flow.
