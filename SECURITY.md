# Security Policy

## Supported Versions

SynchroSonic has not shipped a stable public release yet.

Until the first supported release is tagged:

- the latest default branch is best-effort only
- no long-term support window is promised
- security fixes may land alongside normal development changes

## Reporting A Vulnerability

Please do not open a public GitHub issue for a suspected security problem.

Preferred reporting path:

1. Use GitHub Private Vulnerability Reporting if it is enabled for the repository.
2. If private reporting is not enabled yet, hold the report and contact the maintainers through the private channel published with the release project metadata once that is available.

Current limitation:

- this repository does not yet publish a dedicated security mailbox
- enabling GitHub private reporting or adding a maintainer contact is still a blocker before a broad public release

## What To Include

Please include:

- affected version or commit
- Linux distribution and environment details
- exact reproduction steps
- whether the issue requires local access, LAN access, or no authentication
- logs, traces, or crash output if available
- any workaround you found

## Disclosure Expectations

- give maintainers time to reproduce and assess the issue before public disclosure
- avoid posting exploit details in public issues before a fix or mitigation exists
- when a fix ships, include a concise public note in the release summary
